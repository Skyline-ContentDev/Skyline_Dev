Immediate Response Actions

⏱ 0-2 minutes

Quick Checks in Catalyst Center:

  1. Verify Issue Scope: Check if other devices in same location are also down
  2. Check Maintenance Mode: Look for maintenance window banner in Device 360
  3. Confirm Management Access: Try refreshing the device page to see if it comes back
  4. Note Symptoms: Document what you see for potential escalation

⚠ Critical First Steps

  • Document current device status before taking any actions
  • Verify if this is an isolated incident or part of broader outage
  • Check for maintenance windows that might explain the outage
  • Determine urgency based on device role (access, distribution, core)

💡 Scope Assessment Tips

  • Multiple devices down in same location suggests power or upstream issue
  • Single device offline suggests local hardware or configuration problem
  • Check if issue coincides with recent changes or maintenance
  • Review event timeline for patterns before device went offline

Path A: Complete Connectivity

⏱ 5-10 minutes

⚡ When to Use This Path

  • Path Trace shows complete connectivity to device
  • This suggests the device is reachable but has configuration or SNMP issues

1. Check Device Controllability:

  • Go to Provision > Inventory
  • Change the Focus to Provision
  • Scroll over to see the Manageability status. If Unmanaged is shown, this means that Catalyst Center is unable to gather any information about the device
  • To troubleshoot, navigate to System > Settings > Device Settings > Device Controllability and ensure that Device Controllability is enabled

2. Verify Credentials:

  • In Device 360, look for credential-related errors
  • Navigate to Design > Network Settings > Device Credentials
  • Verify SNMP community strings and CLI credentials are correct

3. Force Device Sync:

  • In Provision > Inventory, choose a device
  • With the device selected, under Actions, choose Inventory, then select Resync Devices
  • Wait for sync to complete and check if device status improves

💡 Configuration Issue Indicators

  • Device responds to ping but not to SNMP queries
  • Authentication failures in event logs
  • Recent configuration changes to SNMP or management access
  • ACL or firewall rules blocking management traffic
  • Wrong VRF for management interface
