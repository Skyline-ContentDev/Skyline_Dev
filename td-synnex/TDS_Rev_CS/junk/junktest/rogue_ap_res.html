Rogue AP Resolution Actions

Comprehensive response workflow for unauthorized access point threats

1

Immediate Response Actions

⏱ 0-2 minutes

Document and Assess:

  1. Take Screenshots of rogue AP details for documentation
  2. Note Key Information: MAC address, SSIDs, location, threat level
  3. Check Company Policy for rogue AP response procedures
  4. Assess Immediate Risk: Determine if containment is needed

⚠ Critical First Steps

  • Capture all evidence immediately - rogue APs may disappear quickly
  • Document timestamp, detecting AP, and signal strength
  • Note any connected clients - this indicates active security breach
  • Verify company policy before taking containment actions

💡 Documentation Tips

  • Screenshot the full rogue details page including all visible information
  • Record the exact time of detection and your response actions
  • Save evidence to secure incident response repository
  • Include floor plan or location map if available
2

High-Risk/Malicious Rogues Response

⏱ 5-10 minutes

1. Immediate Notification:

  • Alert Information Security team via established channels
  • Create incident ticket in ITSM system
  • Document all evidence for potential forensic analysis

2. Containment Actions (if authorized):

  • In Rogue AP details, under Actions, click "Contain" button
  • Document any actions taken

Available Containment Actions:

  • Start Containment: Initiates containment of the BSSID in the environment
  • Stop Containment: Ceases containment of the BSSID in the environment
  • Add to Allowed List: Removes the AP from the active monitoring state as a threat

3. Investigation Coordination:

  • Work with Physical Security to investigate location
  • Coordinate site visit if rogue is on premises
  • Preserve evidence for potential legal action

⚠ Containment Legal Considerations

  • Containment can have legal consequences - verify authorization first
  • Auto-contain should be used with extreme caution
  • If rogue AP is connected to switching infrastructure, switchport shutdown may be an option
  • In shared environmental spaces, false positives are likely until Allowed List is built
  • Document all containment actions and justifications thoroughly

⚡ High-Risk Indicators

  • Corporate SSID spoofing (Honeypot attack)
  • AP Impersonation with same BSSID/ESSID as managed AP
  • Rogue on Wire (connected to internal network)
  • Multiple connected clients from internal network
  • Strong signal strength indicating close proximity
